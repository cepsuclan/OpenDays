1. Executive Summary

Incident Overview

This report details the findings of a Level 2 cyber incident involving advanced evasion techniques and the exfiltration of sensitive academic data.

The attacker successfully gained access using a stolen service account, moved laterally across two internal systems, and utilized a non-standard exfiltration endpoint to steal a large database file, demonstrating an effort to blend in with network noise.

The primary objectives of the intrusion were met: Access, Lateral Movement, and Data Exfiltration.

Key Findings Summary

  • Initial Access: External IP `203.0.113.50` connected successfully via SSH.
  • Compromised Host: The initial machine compromised was the Entry Point at `192.168.1.110`.
  • Lateral Movement: The attacker moved from the Entry Point to the Database Server at `192.168.1.240` using an SQL connection.
  • Stolen Data: The file `Student_Records_Backup.sql` was stolen.
  • Exploited Credential: The user account db_user_backup was compromised.

3. Mitigation and Remediation

3.1 Immediate Actions

The following steps must be taken immediately to contain the breach:

  1. Lock/Disable Compromised Accounts

    Immediately disable or reset the password for the compromised user account: db_user_backup across all systems and review all service accounts for similar privilege levels.

  2. Isolate and Rebuild Servers

    Take the Entry Point (192.168.1.110) and Database Server (192.168.1.240) offline. Do not return them to service until they have been forensically imaged and completely rebuilt from a trusted, clean source.

  3. Block External IP Addresses

    Implement a firewall rule to deny all inbound/outbound traffic to the attacker's endpoint: 203.0.113.50.

3.2 Recommended Long-Term Changes

  1. Implement Multi-Factor Authentication (MFA)

    Mandate MFA for all high-privilege accounts, including all service and administrative users, to prevent credential stuffing attacks.

  2. Restrict Outbound Traffic

    Restrict the Database Server (192.168.1.240) from initiating any unnecessary outbound connections. Data theft relies on moving data out; limiting this greatly reduces the risk of exfiltration.

  3. Review Application/Service Account Privileges

    Audit the db_user_backup account's permissions. This account should not have the ability to run general SSH commands or initiate large outbound transfers.
