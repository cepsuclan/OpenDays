Digital Forensics Incident Report - Level 1 Findings

Incident Date: 2024-10-25

Report Date: 25/10/2024

Report Date (Header): 25/10/2024

Investigator: You

Incident Summary: Unauthorized external access to an internal Jump Host, followed by lateral movement to an RDP server and subsequent exfiltration of confidential financial data.

►

Key Findings (Evidence Chain)

1. Initial Access (External Intrusion)

The attacker first gained a foothold by successfully connecting from an external IP address to an internal host.

Time: 18:00:40 | Source: 172.55.9.123 | Target: 192.168.1.100 | Action: Remote desktop login (successful) to Jump Host (user 'admin_tool')
  • Attacker IP (External): 172.55.9.123
  • Compromised Host (Jump Host): 192.168.1.100
  • Exploited Credential: The credential for user 'admin_tool' was successfully used.

2. Lateral Movement

From the Jump Host, the attacker immediately performed network discovery ('net use') and then moved to a secondary server.

Time: 18:00:50 | Source: 192.168.1.100 | Target: 192.168.1.250 | Action: RDP Session Initiation (User: 'admin_tool')
  • Lateral Target (RDP Server): 192.168.1.250
  • The RDP server was then used to stage the target file, 'Q1_P&L_Final.zip', before exfiltration.

3. Data Exfiltration (Theft)

The attacker then sent the stolen data back to their external IP address from the Jump Host (192.168.1.100).

Time: 18:01:30 | Source: 192.168.1.100 | Target: 172.55.9.123 | Action: Large outbound file transfer: Q1_P&L_Final.zip (25 MB)
  • Stolen File: Q1_P&L_Final.zip
  • Exfiltration Destination: 172.55.9.123 (Attacker's IP)
►

Recommended Immediate Remediation

  1. Block Attacker IP

    Immediately implement a firewall rule to **block all inbound and outbound traffic** to and from the external IP address **172.55.9.123**.

  2. Revoke Compromised Credentials

    The **'admin_tool'** user account must be **locked out and its password changed** across all systems immediately. Review all associated logs for further unusual activity.

  3. Isolate and Rebuild Host

    **Take the Jump Host** (**192.168.1.100**) and **RDP Server** (**192.168.1.250**) **offline**. Do not return them to service until they have been forensically imaged and completely rebuilt from a trusted, clean source.
